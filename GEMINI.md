# GEMINI.md

## 目的

このドキュメントは、Gemini CLI 向けのコンテキストと作業方針を定義します。

## 出力スタイル

- 言語は日本語を使用する。
- 丁寧かつ簡潔なトーンで回答する。
- 重要な決定や変更については理由を添えて説明する。

## 共通ルール

- コミットメッセージは [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) に従い、説明は日本語で行う。
- 日本語と英数字の間には半角スペースを挿入する。

## プロジェクト概要

- PeX の投資案件および定期預金案件をクロールし、Discord へ通知する TypeScript アプリケーション。
- `cheerio` を利用したスクレイピングと、`@book000/node-utils` を利用したユーティリティ機能で構成される。

## コーディング規約

- コード内のコメントは日本語で記載する。
- エラーメッセージは英語で記載する。
- JSDoc コメントは日本語で記載する。
- `skipLibCheck: true` による型チェックの回避は行わない。

## 開発コマンド

```bash
# 依存関係のインストール
pnpm install

# 実行
pnpm start

# テスト
pnpm test

# Lint と自動修正
pnpm lint
pnpm fix
```

## 注意事項

- **機密情報の保護**: API キーや Webhook URL などの機密情報は、絶対にファイルに書き込んだりコミットしたりしない。
- **既存パターンの尊重**: 既存の `ConfigFramework` や `Notified` クラスの設計パターンを尊重して拡張を行う。
- **ドキュメントの整合性**: 設定を変更した場合は、必要に応じて `mkdir -p schema` でディレクトリを作成したうえで `pnpm generate-schema` を実行し、`schema/Configuration.json` を再生成しておくこと。

## リポジトリ固有

- `src/main.ts` がエントリーポイントであり、案件の取得から通知までのメインフローが記述されている。
- `data/` ディレクトリは永続化データ（設定、通知済みフラグ）を保持するために使用される。
